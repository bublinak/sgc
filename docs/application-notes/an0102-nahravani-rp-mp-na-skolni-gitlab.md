## Jak správně odevzdat rok svého života na školní GitLab

Ano, je to třeba. Ne, nemůžete to poslat mailem. Jo, musí to být na školní gitlab. Ne, nestačí to tam nahrát přes web. To readme by to chtělo dopsat.

## Repozitáře a jejich struktura

### Kde repozitář najít
- adresa vašich projektů je centralizovaná s ohledem na aktuální třídu a téma: `https://gitlab.spseplzen.cz/studentske-projekty/projekty-2025-2026/odp/<třída>/<název_projektu>`
- po přihlášení do školního GitLabu uvidíte repozitář připravený přímo pro vás
- při **odevzdávání na větev `main` narazíte na její ochranu** — nesmíte do ní posílat commity přímo

> **Poznámka**: `--force` vám nepomůže.

### Odevzdávací workflow
1. Připravte svůj repozitář k odevzdání (úklid dat, dopsání README, zkontrolování struktury)
2. push na skolní gitlab na větev `odevzdani` 
- jakmile je práce hotová a připravená k odevzdání, otevřete na GitLabu **Pull Request** (Merge Request) z vaší větve `odevzdani` do větve `main`

#### Vytvoření větve v terminálu
- založení a přesun: `git checkout -b odevzdani`
- nahrání nově vzniklé větve na server: `git push -u origin odevzdani`

> A nebo taky přes grafické rozhraní. Pokud používáte vývojové prostředí, jako je VS Code, stačí kliknout vlevo dole na název větve (`main`), vybrat "Create new branch from...", nazvat ji `odevzdani` a využít tlačítka "Publish Branch". Merge Request pak "naklikáte" přímo ve webovém rozhraní GitLabu.

## Struktura odevzdávané práce

Předzaložený repozitář obsahuje základní `README.md` a složky pro odevzdání. Aby byla struktura pro fork kompatibilní, musí složky odpovídat cílovému repozitáři, na který se bude práce nahrávat.

```text
REPO
 |
 +- README.md
 +- dokumentace 
 |   +- DMP_<příjmení>_<název_práce>.pdf
 |   +- PRILOHA_<římskymi_cislicemi>_<název>.pdf
 +- reseni
 |   +- <zdrojové_kódy_nebo_výsledky_práce>
 |   +- PRILOHA_<římskymi_cislicemi>_<název>  #třeba webová aplikace
 |       +- <vlastní soubory aplikace>
     +- <projektová_data_a_podklady>
 +- projektove_podklady
     +- <projektová_data_a_podklady>
```
#### README.md
- základní popis projektu, jeho cílů, použitých technologií a stručný návod k reprodukci výsledků
- může obsahovat odkazy na důležité soubory v repozitáři
- měl by mít rychlý návod ke spuštění pokud možno

#### dokumentace
- složka určená pro samotný text práce
- povinně zde musí ležet finální exportované PDF pojmenované přesně podle pravidla `DMP_<příjmení>_<název_práce>.pdf`
- dále sem patří případné přílohy v PDF (výkresy, schémata, tabulky, ...)

#### reseni
- dřívější "přílohy". Sem patří samotný výsledek práce z praktického hlediska (zdrojové kódy, výsledky měření, grafy, ...). Vše, co je potřeba k tomu, aby někdo jiný mohl práci zreprodukovat.
- je vhodné udržovat zdrojové kódy či vývojové celky v logicky pojmenovaných podsložkách
- pokud přikládáte 

#### projektove_podklady
- určeno pro doprovodná pracovní data z výzkumu a vývoje: soubory 3D modelů (.stl, .step), měření z laboratoří, grafy či pomocné skripty z průběhu realizace, ...

## Nahrání a odevzdání práce
1. Připravte repozitář podle výše uvedené struktury a pravidel pojmenování
```bash
git add .
git commit -m "Příprava práce k odevzdání"
```
2. Pushněte svou práci na školní GitLab do větve `odevzdani`
```bash
git push https://gitlab.spseplzen.cz/studentske-projekty/projekty-2025-2026/odp/<třída>/<název_projektu> odevzdani
```
3. Otevřete Pull Request (Merge Request) z větve `odevzdani` do `main` na školním GitLabu (vaše repo -> </> Code -> Merge Requests -> New Merge Request)
   - Source branch: `odevzdani`
   - Target branch: `main` (**MUSÍTE VYBRAT VÁŠ REPOZITÁŘ!!!**, hledejte podle názvu své práce)
   - DÁLE
   - Title: `Odevzdání <RP/MP>`, Asignee: <váš_vedoucí>
   - Create Merge Request

> Pokud merge request můžete rovnou spojit do `main`, udělejte to.
> Po odeslání merge requestu raději informujte svého vedoucího.
